## 摘要
**1) 一句话总结**
Toonflow 是一款基于 AGPL-3.0 协议开源的 AI 短剧全流程创作工具，能够结合第三方 AI 服务将小说文本高效转化为剧本、分镜并最终合成视频。

**2) 核心要点**
* **开源与版本**：采用 AGPL-3.0 开源协议，当前最新版本为 v1.0.6。
* **核心定位**：主打“0门槛全流程AI化”，旨在提升短剧制作、小说影视化改编及剧本原型开发的效率。
* **四大核心功能**：
  * **角色生成**：从小说文本自动提取并生成角色详细设定。
  * **剧本生成**：将事件或章节转化为包含对白与场景的结构化剧本。
  * **分镜制作**：自动生成分镜提示词与画面设计蓝图。
  * **视频合成**：集成 AI 图文视频技术生成片段，并支持在线编辑。
* **外部服务依赖**：系统运行必须接入第三方 AI 接口，包括大语言模型（LLM）、视频生成（Sora 或豆包）以及图片生成（Nano Banana Pro）。
* **部署方式**：支持 Windows 本地安装、Docker 部署（支持数据持久化），以及基于 Node.js（推荐 24.x）和 PM2 的云端部署（适用于 Ubuntu 20.04+ 或 CentOS 7+）。
* **初始凭证**：系统首次登录的默认账号为 `admin`，密码为 `admin123`。

**3) 风险与不足**
* **操作系统限制**：目前本地安装包仅支持 Windows 系统，尚未发布 Linux 与 macOS 版本。
* **核心能力依赖**：软件本身不内置生成模型，图文及视频的生成完全依赖于用户自行配置的第三方 AI 服务接口。

## 功能与定位
Toonflow 面向“短剧/短视频内容生产”的一站式流程：把文本（小说/章节/事件）作为输入，按“角色设定 → 剧本 → 分镜 → 图像/视频生成 → 合成与编辑”的路径组织工作台与任务模块，目标是把影视化改编的生产链条做成可重复执行的流水线。

## 典型使用场景
- 把小说或长文本快速改编为短剧脚本与分镜，作为创作原型或内容验证。
- 批量产出短视频素材：从统一的人物设定与分镜出发生成多段视频片段。
- 团队化生产：把角色/分镜/任务队列拆分成可协作的环节（结合你自己的模型与资产管理体系）。

## 核心功能
- 角色生成：从文本抽取角色信息，并结构化为可复用的角色设定。
- 剧本生成：将章节或事件转换为包含对话与场景描述的结构化脚本。
- 分镜制作：自动生成分镜提示词与镜头画面描述，作为后续图像/视频生成输入。
- 视频合成与编辑：将生成片段整合，并提供编辑与输出流程。

## 使用方式概览
- 前置依赖：需要自行准备并接入 LLM、视频生成与图片生成等外部服务接口。
- 本地体验：可从 GitHub Releases 下载客户端（当前以 Windows 为主）。
- 自托管：支持 Docker Compose 部署；也提供基于 Node.js + PM2 的服务器部署方式。

## 限制与注意事项
- 默认账号密码为公开固定值，部署后应立即修改并限制暴露面（尤其是公网部署）。
- 生成成本与稳定性取决于你接入的第三方模型与服务，生产使用前应先做小规模端到端验证。
