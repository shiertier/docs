## 摘要

### 1) 一句话总结
Nuclei 是一款基于 YAML 模板的高性能漏洞扫描器，支持多协议和大规模自动化测试，致力于通过模拟真实攻击步骤实现极低的误报率。

### 2) 关键点
* **项目定位**：主要作为独立的开源命令行（CLI）工具使用（GitHub Stars 27158），同时提供免费云端服务及 Pro/Enterprise 企业托管版本。
* **模板系统**：基于简单的 YAML 格式创建自定义漏洞检测场景，由社区驱动维护，支持按标签、作者、严重程度和协议类型进行过滤执行。
* **多协议支持**：支持 TCP、DNS、HTTP、SSL、WHOIS、JavaScript 和 Code 等多种协议。
* **高性能与准确率**：具备并行扫描和请求聚类（Clustering）机制，通过模拟真实世界的攻击步骤来验证漏洞，目标是实现零误报。
* **输出与集成**：支持 JSONL、Markdown、SARIF 等输出格式，可无缝集成至 CI/CD 流水线以及 Jira、Splunk、GitHub、GitLab 等平台。
* **企业版特性**：Pro/Enterprise 版本提供 50 倍扫描提速、SAML SSO、团队共享工作空间，且符合 SOC 2 合规标准。
* **技术依赖与高级功能**：安装需 Go >= 1.24.2，支持代理配置、速率限制、Fuzzing 测试以及 Interactsh（OAST 测试）等高级功能。

### 3) 风险与不足
* **版本兼容性风险**：项目处于活跃开发阶段，版本发布可能会包含破坏性变更（Breaking changes），更新前需查阅发布日志。
* **部署安全风险**：Nuclei 主要设计为独立的 CLI 工具，官方明确指出将其作为后台服务（Service）运行会带来安全风险，需谨慎操作并增加额外防护。

## 功能与定位
Nuclei 是一款现代化的、高性能的漏洞扫描工具。它基于简单的 YAML 模板运行，允许用户设计模拟真实环境的自定义漏洞检测场景。项目主要作为独立的命令行（CLI）工具使用，同时也提供免费的云端基础服务，以及面向安全团队和企业的 Pro/Enterprise 云托管版本。

## 典型使用场景
* **自动化安全测试**：集成到 CI/CD 流水线中，用于持续的漏洞检测和回归测试。
* **大规模漏洞扫描**：供渗透测试人员、安全团队和企业在复杂环境中进行大规模、高准确率的持续漏洞扫描。

## 核心功能
* **多协议支持**：支持 TCP、DNS、HTTP、SSL、WHOIS、JavaScript、Code 等多种协议的扫描。
* **高性能处理**：具备超快速的并行扫描处理能力和请求聚类（Clustering）机制。
* **灵活的模板系统**：使用简单的 YAML 格式创建和自定义漏洞检测模板，支持按标签、作者、严重程度、协议类型等进行精细化过滤和执行。
* **丰富的输出与集成**：
  * 支持 JSONL、Markdown、SARIF 等多种结果输出格式。
  * 可与 Jira、Splunk、GitHub、Elastic、GitLab 等平台集成。
* **云端扩展能力**：提供云端存储与可视化漏洞结果、模板管理及目标发现功能（企业版进一步支持 AWS、GCP、Azure 等云服务及 Slack、Webhooks 集成）。

## 特色与差异点
* **极低误报率**：通过模拟真实世界的攻击步骤来验证漏洞，致力于实现零误报（Zero false positives）。
* **社区驱动的模板库**：由数千名安全专业人员共同贡献模板，能够快速响应和覆盖最新趋势的漏洞。
* **企业级性能与合规**：Pro 和 Enterprise 版本在开源版基础上进行了优化，提供 50 倍的扫描速度提升、实时扫描、SAML SSO、团队共享工作空间，且平台符合 SOC 2 合规标准（提供欧盟和美国托管选项）。

## 使用方式概览
* **环境依赖**：安装需要 Go >= 1.24.2。
* **安装方式**：通过 `go install -v github.com/projectdiscovery/nuclei/v3/cmd/nuclei@latest` 获取。
* **命令行调用**：通过 `nuclei [flags]` 执行扫描。支持丰富的参数配置，涵盖目标输入、模板选择、结果过滤、网络配置（如代理、重定向控制、速率限制）、Fuzzing 测试以及 Interactsh（用于 OAST 测试）等高级功能。

## 限制与注意事项
* **版本兼容性**：项目处于活跃开发阶段，版本发布可能会包含破坏性变更（Breaking changes），建议在更新前查阅发布日志。
* **部署风险**：Nuclei 主要设计为独立的 CLI 工具。将其作为后台服务（Service）运行可能会带来安全风险，官方建议谨慎操作并采取额外的安全防护措施。
