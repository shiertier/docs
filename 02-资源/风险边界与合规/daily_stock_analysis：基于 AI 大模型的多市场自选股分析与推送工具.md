## 摘要

**1) 一句话总结**
这是一个基于 AI 大模型的开源股票智能分析系统，支持多市场自选股的自动化分析、策略问答，并能生成包含具体买卖点位的决策仪表盘推送到多种通讯工具。

**2) 核心要点**
*   **基础信息**：项目基于 Python 3.10+ 开发，采用 MIT 开源协议，目前拥有 12650 颗 Stars。
*   **支持市场与输出**：支持 A股、港股、美股及美股指数，每日生成「AI 决策仪表盘」，提供一句话核心结论、精确买卖点位（买入、止损、目标价）及操作检查清单。
*   **AI Agent 与策略**：内置 11 种交易策略（如缠论、波浪理论、均线金叉等），支持多轮自然语言对话问股，并提供历史策略回测功能以验证胜率。
*   **多维度数据与模型兼容**：整合技术面、筹码分布、舆情与实时行情；兼容 DeepSeek、Claude、OpenAI、Gemini 等主流大模型，以及 AkShare、YFinance、Tavily 等多种行情与新闻搜索源。
*   **视觉识别添加**：支持通过 Vision LLM 自动识别持仓 APP 或行情截图，快捷提取股票代码并加入监控池。
*   **多渠道推送**：支持将分析结果自动化推送到企业微信、飞书、Telegram、钉钉、邮件、Server酱等主流渠道。
*   **部署方式**：深度适配 GitHub Actions 实现零成本免服务器定时运行，同时支持本地运行、Docker 部署以及提供可视化 Web 界面。
*   **内置风控**：系统自带交易纪律规则，如乖离率超阈值自动提示风险（严禁追高）、要求均线多头排列，并可配置新闻时效过滤。

**3) 风险与不足**
*   **免责声明**：项目仅供学习和研究使用，不构成任何投资建议，作者不对使用产生的任何损失负责。
*   **API 费用成本**：Agent 策略问股模式依赖外部大语言模型，多轮对话会产生相应的 API 调用费用。
*   **图片识别限制**：从图片添加股票的功能必须配置具备 Vision 能力的模型，且单张图片最大限制为 5MB。
*   **安全风险**：若在 Webhook 配置中关闭 HTTPS 证书校验（`WEBHOOK_VERIFY_SSL=false`）会存在严重安全风险，仅建议在可信内网环境中使用。

## 功能与定位
本项目是一个基于 AI 大模型的自选股智能分析系统，支持 A股、港股、美股及美股指数。其核心定位是通过自动化脚本或本地服务，每日整合多维度市场数据与 AI 分析能力，生成包含操作建议的「决策仪表盘」，并推送到用户的常用通讯工具中，以辅助投资决策。

## 典型使用场景
- **每日复盘与监控**：每日定时获取大盘概况、板块涨跌，以及自选股的买卖点位和技术面分析。
- **策略深度问答**：通过自然语言与 AI Agent 对话，使用特定策略（如缠论、波浪理论）对个股进行深度剖析。
- **快捷添加自选股**：直接上传持仓 APP 或行情列表的截图，系统自动提取股票代码并加入监控池。
- **历史策略验证**：通过 AI 回测功能，评估历史分析的准确率、方向胜率及止盈止损命中率。

## 核心功能
- **AI 决策仪表盘**：输出一句话核心结论、精确的买卖点位（买入价、止损价、目标价）以及操作检查清单。
- **多维度数据整合**：结合技术面、筹码分布、舆情情报与实时行情进行综合分析。
- **Agent 策略问股**：内置 11 种交易策略（如均线金叉、缠论等），支持多轮对话，实时展示 AI 从获取行情到生成结论的思考路径。
- **图片识别添加**：利用 Vision LLM 自动识别并提取图片中的股票代码。
- **多渠道自动化推送**：支持推送到企业微信、飞书、Telegram、钉钉、邮件、Pushover、PushPlus 及 Server酱等渠道。

## 特色与差异点
- **内置交易纪律**：系统自带风控规则，如乖离率超阈值自动提示风险（严禁追高）、要求均线多头排列（趋势交易），并可配置新闻最大时效以过滤过时信息。
- **零成本免服务器部署**：深度适配 GitHub Actions，用户只需配置 Secrets 即可实现每日定时自动运行。
- **广泛的模型与数据兼容性**：支持 Gemini、Claude、OpenAI 兼容模型、DeepSeek、通义千问等主流大模型；整合 AkShare、YFinance 等行情源及 Tavily、SerpAPI 等新闻搜索源。
- **可视化 Web 界面**：提供完整的配置管理、任务监控和手动分析功能，支持密码保护敏感配置。

## 使用方式概览
- **GitHub Actions 部署（推荐）**：Fork 仓库后，在设置中配置 AI 模型 API Key 及推送渠道的 Secrets，启用 Actions 即可在每个工作日定时执行。
- **本地或 Docker 部署**：克隆项目并安装依赖（`requirements.txt`），配置 `.env` 环境变量文件后运行主程序。
- **Web 界面运行**：编译前端资源后，通过启动命令开启本地 Web 服务，在浏览器中进行可视化操作与 Agent 对话。
