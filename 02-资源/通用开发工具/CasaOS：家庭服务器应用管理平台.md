## 摘要

**一句话总结**
CasaOS 是一款专为家庭场景设计的开源个人云系统，通过极简的界面和广泛的软硬件兼容性，帮助用户低成本地将闲置设备改造为个人数据中心，并轻松管理海量 Docker 应用。

**关键要点**
* **项目定位**：旨在解决消费者数据资产所有权问题，帮助用户建立数据自主权，降低 SaaS 订阅成本，并为未来训练个性化 AI 助手提供数据基础。
* **典型场景**：适用于搭建家庭数据中心（供创作者和小型组织协作）、智能家居中控，以及闲置设备（旧电脑、开发板等）的再利用。
* **应用生态**：内置精选应用商店，支持一键安装 Nextcloud、HomeAssistant 等常用服务，并兼容超 10 万款 Docker 应用。
* **用户体验**：采用无代码、无表单的直观 UI 设计，提供“所见即所得”的文件与磁盘管理，以及可视化的系统与应用状态看板，零技术门槛。
* **硬件兼容**：支持 amd64/x86-64、arm64 和 armv7 架构，兼容 ZimaBoard、Intel NUC、树莓派及各类旧电脑。
* **系统支持**：官方推荐使用 Debian 12，同时官方测试支持 Ubuntu Server 20.04 和 Raspberry Pi OS。
* **部署与维护**：支持一行终端命令快速安装，提供专用的卸载命令，并支持通过 Web UI 一键更新或终端命令行更新。

**风险与不足**
* **更新操作限制**：若选择通过命令行更新系统，必须通过 SSH 远程或直接连接物理设备的键鼠显示器进行，**严禁**在 CasaOS Web UI 内置的终端面板中执行更新命令。
* **部分系统未经官方验证**：在 Alpine、OpenWrt 和 ArchLinux 等操作系统上目前仅由社区提供初步支持，尚未经过官方的全面测试。

## 功能与定位

CasaOS 是一款简单、易用、优雅的开源个人云系统。该项目旨在解决消费者数据资产所有权问题，通过提供低成本的个人数据中心，帮助用户建立数据自主权，降低 SaaS 订阅成本，并为未来训练个性化 AI 助手提供数据基础。

## 典型使用场景

* **家庭数据中心**：为创作者和小型组织提供低成本的数据存储、管理与协作方案。
* **智能家居中控**：控制并连接各类智能设备，提供跨生态的本地智能服务。
* **闲置设备改造**：将旧电脑、开发板等闲置硬件转化为功能完善的个人云服务器。

## 核心功能

* **应用商店与一键安装**：内置精选应用商店，支持一键安装 Nextcloud、HomeAssistant、AdGuard、Jellyfin 等常用服务。
* **海量 Docker 应用支持**：能够轻松安装和运行来自 Docker 生态的超 10 万款应用。
* **直观的文件与磁盘管理**：提供“所见即所得”的优雅文件管理界面，无需任何技术背景即可轻松上手。
* **系统与应用看板**：通过精心设计的小组件（Widgets），一目了然地展示系统资源占用、应用运行状态等核心信息。

## 特色与差异点

* **专为家庭场景设计**：采用无代码、无表单的直观 UI 设计，极大降低了服务器系统的使用门槛。
* **广泛的硬件兼容性**：支持 amd64/x86-64、arm64 和 armv7 架构，兼容 ZimaBoard、Intel NUC、树莓派（Raspberry Pi）以及各类旧电脑。
* **多操作系统支持**：
  * **官方推荐与测试**：Debian 12（推荐）、Ubuntu Server 20.04、Raspberry Pi OS。
  * **社区支持**：Elementary 6.1、Armbian 22.04 等。
* **项目起源**：“Casa”在西班牙语中意为“家”。该项目最初作为众筹硬件 ZimaBoard 的预装系统诞生，旨在填补市场上缺乏专为家庭场景设计的服务器系统的空白。

## 使用方式概览

* **快速部署**：在全新安装的兼容操作系统上，通过一行终端命令即可完成 CasaOS 的快速安装。
* **系统更新**：支持直接通过 CasaOS 的 Web UI 设置进行一键更新；也可通过终端命令行进行更新。
* **系统卸载**：官方提供了专用的卸载命令，方便用户随时移除系统。

## 限制与注意事项

* **终端更新限制**：若选择通过命令行更新 CasaOS，必须通过 SSH 远程连接或直接连接物理设备的键盘与显示器进行操作，**不能**在 CasaOS Web UI 内置的终端面板中执行更新命令。
* **部分系统兼容性未经验证**：Alpine、OpenWrt 和 ArchLinux 等操作系统目前仅由社区提供初步支持，尚未经过官方的全面测试。
