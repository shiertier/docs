## 摘要

**1) 一句话总结**
Structurizr Playground 是一个开箱即用的浏览器内演练环境，允许用户通过编写 Structurizr DSL 快速生成 C4 架构图，并支持在线试用或本地化部署。

**2) 核心要点**
- **工具定位**：专为 C4 模型设计的“模型即代码（models as code）”DSL 独立演练平台。
- **核心功能**：实时解析用户编写的 Structurizr DSL 并自动渲染为对应的架构图示。
- **零配置入门**：提供官方托管的在线实例，无需安装任何工具即可快速试用和编写架构草稿。
- **文件交互**：支持直接从本地计算机上传 DSL 文件至演练环境中。
- **缓存管理**：支持将 DSL 代码保存至浏览器本地存储或从中加载历史记录。
- **主题与样式**：提供主题浏览器功能，方便用户管理和应用图表样式。
- **本地化部署**：除在线 Web 版本外，官方还提供预构建的 Docker 镜像和 Java WAR 文件供用户在本地独立运行。
- **Docker 专属特性**：Docker 镜像版本已直接打包预构建的主题，用户可通过环境变量启用。

## 功能与定位

Structurizr Playground 是 Structurizr 官方提供的独立 DSL 演练环境，面向希望用文本方式快速描述软件系统结构的用户。Structurizr 官方首页把整个产品定位为面向 C4 model 的 models as code 工具，而 Playground 则把这一能力压缩成一个浏览器可直接试用的入口，重点放在编写 Structurizr DSL、解析 DSL 和渲染架构图示上。

相较于自由画布或通用白板工具，它更偏向“先写模型，再出图”的工作方式，适合已经接受文本建模思路、希望快速验证系统结构表达是否清晰的用户。

## 典型使用场景

- 用于第一次接触 Structurizr DSL 时的在线试用，不必先安装本地工具链。
- 用于快速起草 C4 架构图，把系统边界、容器关系或组件关系先写成可讨论的文本草稿。
- 用于把已有 DSL 文件导入浏览器，快速检查渲染结果、主题样式或建模表达是否合理。
- 用于团队内部演示、教学或短时工作坊，在浏览器里直接展示“文本到架构图”的流程。
- 用于需要本地控制环境的场景，按官方文档在本机独立运行 Playground，再继续做 DSL 演练。

## 核心功能

- 支持从本机上传 DSL 文件，直接进入解析和渲染流程。
- 支持从浏览器本地存储加载历史 DSL 内容，也支持把当前 DSL 保存回本地存储。
- 支持打开 theme browser，查看和切换图示主题。
- 支持解析 Structurizr DSL，并把文本模型渲染为对应的架构图示。
- 提供官方托管实例，方便零安装试用。
- 提供本地运行路径，官方文档给出 Docker 镜像和 Java WAR 两种入口。

## 特色与差异点

- 它不是通用画图板，而是围绕 Structurizr DSL 和 C4 建模方式构建的专用工具，适合文本驱动的架构表达。
- 在线实例把试用门槛压得很低，适合先验证建模思路，再决定是否继续接入更完整的 Structurizr 工具链。
- 官方同时保留本地运行方式，让用户可以在受控环境里继续使用 Playground，而不必被在线实例绑定。
- 根据官方页面列出的功能范围推断，Playground 的重点是 DSL 演练、主题试用与图示渲染，而不是完整的团队协作或长期工作区治理能力。

## 使用方式概览

1. 直接访问官方在线实例，在浏览器中编写或上传 Structurizr DSL，立即查看渲染结果。
2. 需要保留短期草稿时，可使用浏览器本地存储的加载与保存能力。
3. 需要本地运行时，参考官方 Playground 文档提供的 Docker 镜像或 Java WAR 方案自行部署。
4. 若后续需求扩展到长期维护、更多文档能力或更完整的产品功能，再继续评估整个 Structurizr 文档体系。

## 限制与注意事项

- 官方页面公开强调的是 Playground 作为独立 DSL 演练环境的能力，不应把它直接等同于 Structurizr 全产品线。
- 官方列出的保存方式主要是本地文件和浏览器本地存储；如果你需要团队协作、长期托管或更完整的治理能力，应继续查看 Structurizr 其他产品与文档。
- 本仓库仅记录可验证的公开信息；本地运行的具体命令、环境变量和端口细节请以官方文档为准。
