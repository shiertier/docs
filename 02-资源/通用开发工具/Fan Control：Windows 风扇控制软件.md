## 摘要

**1) 一句话总结**
Fan Control 是一款面向 Windows 的低资源占用风扇控制软件，通过统一的配置与混合曲线系统，实现对 CPU、GPU 及机箱等整机散热设备的集中定制与管理。

**2) 核心要点**
*   **软件定位**：主打“低资源占用 + 高度可定制”，主程序闭源，底层依赖 LibreHardwareMonitor、NvAPIWrapper 等库读取传感器与执行控制。
*   **统一管理**：将主板、显卡、水冷等风扇收敛至同一套控制体系，避免多套软件冲突，并支持在安静/性能等多套配置文件间快速切换。
*   **混合曲线与传感器**：支持基于 CPU、GPU、主板、硬盘等多个温度源制定曲线，核心特色是支持“混合”风扇曲线与传感器输入，按整机热量响应。
*   **基础控制与校准**：提供首次启动引导，支持手动控制、速度传感器配对、风扇校准以及自定义图形曲线。
*   **插件生态**：内置 .NET 插件系统，允许通过社区插件扩展对外部设备和额外传感器的支持。
*   **获取与安装**：可通过 GitHub Release 下载压缩包或安装程序，也支持通过 Scoop 与 Winget 命令行工具安装。

**3) 风险与不足**
*   **显卡控制限制**：部分 NVIDIA/AMD 显卡在最低转速百分比、0 RPM 触发条件等方面存在硬件或驱动层面的客观约束。
*   **杀毒软件误报**：历史版本的某些驱动组件曾被 Windows Defender 标记为木马并导致传感器不可用（较新版本已调整发布策略与依赖）。
*   **硬件兼容性依赖**：硬件兼容能力高度依赖其底层的硬件监控开源库，部分兼容性问题源于底层库而非软件本体。

## 功能与定位

Fan Control 是面向 Windows 的风扇控制软件，主打“低资源占用 + 高度可定制”的统一控制体验：把 CPU、GPU 与机箱/水冷等风扇的控制尽量收敛到同一套配置与曲线体系里。

GitHub 发布仓库说明该软件的源代码为闭源；同时网站与说明文档列出了其依赖的硬件监控/厂商接口相关库（例如 LibreHardwareMonitor、NvAPIWrapper、ADLXWrapper），用于读取传感器并执行控制。

## 典型使用场景

- 统一管理主板风扇、显卡风扇与 AIO 等设备，减少同时使用多套控制软件造成的“各管各的”。
- 为不同负载状态设置多套配置文件，在安静/性能等策略之间快速切换。
- 根据 CPU、GPU、主板、硬盘等多个温度源制定曲线，并用“混合曲线/混合传感器”的逻辑让机箱风扇按系统整体热量来响应。

## 核心功能

- 控制与校准：支持手动控制或将风扇绑定到曲线；支持速度传感器配对与风扇校准。
- 曲线系统：提供多种曲线类型（包含自定义图形曲线与 Mix Fan Curve），并支持以不同函数对多条曲线进行混合。
- 多温度源：支持来自 CPU、GPU、主板、硬盘等的多个温度来源。
- 配置与体验：首次启动引导式设置；支持保存/编辑/加载多个配置文件；支持主题与配色。
- 插件扩展：提供插件系统，允许以 .NET 插件向软件注入额外的传感器或控制能力。

## 特色与差异点

- “混合”能力是其强调的核心卖点之一：不仅能混合风扇曲线，也能混合传感器输入，以更贴近整机散热目标。
- 闭源主程序 + 可扩展生态：发布仓库明确说明主程序闭源，但通过插件机制与社区插件扩展外部设备与传感器支持。

## 使用方式（概览）

- 安装与启动：发布仓库说明可从 Release 页面下载压缩包或安装程序，启动 `FanControl.exe`；并提到可通过 Scoop 与 Winget 安装。
- 配置流程：按文档完成速度传感器配对、校准、曲线设定与温度源绑定，再将曲线应用到对应控制项。

## 限制与注意事项

- 显卡控制存在客观限制：文档提到部分 NVIDIA/AMD 显卡在最低转速百分比、0 RPM 触发条件等方面存在硬件/驱动层面的约束。
- 杀毒软件误报与驱动依赖变更：发布仓库公告与警告提到历史版本的某些驱动组件被 Windows Defender 标记为木马并导致传感器不可用，并说明较新版本的发布策略与依赖有所调整。
- 硬件兼容性很大程度依赖底层硬件库：发布仓库将硬件兼容问题与所依赖的硬件监控项目关联，并建议区分“软件本体问题”与“底层库兼容问题”。
