## 摘要

**一句话总结**
Anthropic 为其命令行编码工具 Claude Code 推出了“自动记忆”（Auto Memory）功能，通过自主记录并加载项目上下文，大幅减少开发者重复输入背景信息的负担。

**关键要点**
* **功能定位**：Claude Code 上线“自动记忆”功能，旨在提升 AI 在长期项目协作中的上下文持续理解能力。
* **记录内容**：在日常工作中自动记录构建命令、调试经验、代码风格偏好及架构约定等关键信息。
* **文件机制**：新增由 Claude 自主维护的 `MEMORY.md` 入口文件，与开发者手动提供指令的 `CLAUDE.md` 文件形成互补。
* **存储位置**：每个项目的自动记忆位于本地目录 `~/.claude/projects/<project>/memory/` 下。
* **加载策略**：每次会话启动时自动加载 `MEMORY.md` 的前 200 行以确保关键上下文即时生效，更长内容则按需读取以平衡性能。
* **管理配置**：该功能默认开启，开发者可通过输入 `/memory` 命令或修改配置文件来进行管理或关闭。
* **核心价值**：解决 AI 编程工具“会话即失忆”的问题，使模型转变为持续参与项目的虚拟成员，从而提升整体开发效率。

## 正文

Anthropic 为 Claude Code 引入了自动记忆（Auto Memory）功能，目标是在长期项目协作中持续积累上下文，减少开发者在每次新会话里重复解释项目背景、约束和个人偏好的成本。

开启后，Claude 会根据协作过程中的纠正和偏好，自动记录构建命令、调试经验、代码风格、架构约定等信息。这些内容会在后续会话中继续被读取和使用，让工具逐步形成对当前项目的持续理解。

这套机制与 `CLAUDE.md` 形成互补。`CLAUDE.md` 仍然是开发者手动编写的项目指令文件，用来明确规则、约束和工作方式；自动记忆则由 Claude 自行维护，用来沉淀它在实际协作中学到的模式、偏好和经验。

根据 Claude Code 官方文档，每个项目都会有独立的自动记忆目录，位于 `~/.claude/projects/<project>/memory/`。其中 `MEMORY.md` 是入口文件，用于索引更细分的主题记忆；同一 Git 仓库下的 worktree 和子目录会共享这份自动记忆，Git 仓库之外则按项目根目录区分。

在加载策略上，Claude 会在每次会话开始时先读取 `MEMORY.md` 的前 200 行，超过部分不会在启动阶段全部塞入上下文，而是在需要时按需读取更细的记忆文件。这种做法兼顾了上下文可用性、性能和成本，也避免长期记忆无限膨胀后挤占当前任务的有效上下文。

该功能默认开启，开发者可以通过 `/memory` 查看、编辑和切换自动记忆。对日常开发来说，这意味着 Claude Code 不再只是一次性问答助手，而会逐步记住团队偏好、常用命令和项目约定，减少重复沟通并提高协作效率。
