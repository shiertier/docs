# Agent Builder 全新升级：全局对话、文件上传与工具管理中心

## 文档信息
- 来源：https://blog.langchain.com/new-in-agent-builder-all-new-agent-chat-file-uploads-tool-registry/
- 发布日期：2026-02-18

## 摘要
**1) 一句话总结**
LangSmith Agent Builder 推出重大更新，新增支持全工具调用的全局对话、一键生成专属智能体、文件上传及集中式工具管理功能，旨在让智能体像团队伙伴一样高效协作。

**2) 核心要点**
*   **全局对话智能体**：新增全局对话（Chat）功能，可直接访问工作区连接的所有工具（如 Slack、Gmail、Linear、Pylon 及远程 MCP 服务器），无需预先配置即可处理临时任务。
*   **自动规划与审批机制**：智能体能自动制定计划并串联调用多个工具，在执行关键操作（如发送消息或创建工单）前会主动请求用户批准。
*   **多线程并发处理**：支持同时运行多个独立的对话线程，用户可随时切换以查看进度、批准操作或追加问题。
*   **一键转化为专属智能体**：用户只需点击按钮，即可将自然语言对话过程转化为可复用的专属智能体，无需进行提示词工程或处理复杂逻辑。
*   **多维度的智能体触发方式**：生成的专属智能体支持手动触发、设定定时任务，或基于外部事件（如 Slack 消息、电子邮件）自动响应。
*   **新增文件上传能力**：支持在任意对话中上传 CSV、屏幕截图或文本文件，从而实现即时数据分析、图像处理（如照片转文档）及参考资料（如提示词、指南）导入。
*   **一站式工具管理中心**：在工作区设置中集中展示所有已连接工具，支持新工具身份验证、添加远程 MCP 服务器以及失效工具的重新验证提示。
*   **严格的权限治理**：为保障团队安全，仅限工作区管理员添加新工具，确保工具访问权限始终受控。

## 正文
今天，我们对 LangSmith Agent Builder 进行了重大更新，核心理念非常简单：**让智能体用起来就像你的团队伙伴一样。**

我们围绕这一理念重构了 Agent Builder。现在，你可以通过一个随时待命的“全局对话（Chat）”智能体来处理临时任务或创建专属智能体。此外，我们还新增了文件上传功能，并让智能体可使用的工具变得一目了然。

### 聚合所有工具的全局对话智能体

Agent Builder 的全局对话现在可以访问你连接到工作区的所有工具（如 Slack、Gmail、Linear、Pylon 以及通过远程 MCP 服务器连接的任何工具）。这意味着你可以直接提问（例如“我有哪些未解决的 Linear 工单？”或“总结今天 #support 频道的需求”），而无需事先设置专门的智能体。

输入问题或需求后，Agent Builder 会制定计划、调用合适的工具并逐步完成任务。如果需要你的批准（如发送消息或创建工单），它会在执行前与你确认。

它还能根据需求串联多个工具。例如，你可以让它快速总结最近的 10 封邮件，然后为其中的会议邀请创建日历事件。Agent Builder 会根据需要自动调用新工具。

此外，你还可以同时运行多个对话。在一个对话中启动任务后，你可以开启新线程处理其他事务。每个对话独立运行，你可以随时切换回来查看进度、批准操作或提出后续问题。

### 一键将对话转化为专属智能体

全局对话功能带来了一种强大的智能体创建方式。你可以从一个自然语言指令开始，例如：“总结本周 Pylon 的支持工单，并为团队起草一份 Slack 更新。” Agent Builder 会处理该任务，你可以通过反馈来不断优化它的输出结果。

在任何时候，只需点击“将此对话转化为可复用的智能体”，即可为该任务生成一个专属的、可重复使用的智能体。

就是这么简单。无需提示词工程，也无需处理复杂的条件逻辑，你的对话过程就是设置过程。所有对话都会被保存，方便你随时回顾并将其转化为智能体，即使你一开始并没有这个打算。

我们发现这种模式在以下场景中表现尤为出色：

*   **研究：** “从网上查找我前十大客户的最新消息，并在 Slack 中发送总结。”
*   **写作：** “回顾公司博客最近的 20 篇文章，并在 Google 文档中制定一份品牌语调指南。”
*   **总结：** “提取本周的 Linear 问题，分析趋势并起草总结。”
*   **沟通：** “为 HubSpot 中的每个新潜在客户研究其公司背景，并起草个性化的开发邮件。”

对于你创建的任何智能体，你都可以手动触发、设定定时任务，或让它们根据外部事件（如 Slack 消息或电子邮件）做出响应。由于它们源于你真实的对话，因此它们已经明确知道什么样的输出才是符合你要求的。

### 支持在任意对话中上传文件

以前，Agent Builder 只能处理来自已连接工具的数据和你在聊天中输入的文本。现在，你可以直接上传 CSV、屏幕截图或文本文件等，让智能体对其进行处理。这带来了以下新能力：

*   **即时数据分析：** 上传 CSV（如上季度的销售数据），让 Agent Builder 发现趋势并将报告发送到 Slack 团队频道。
*   **图像处理：** 上传屏幕截图或照片，让 Agent Builder 进行处理（例如将白板笔记的照片转换为 Google 文档）。
*   **导入文档和参考资料：** 上传写作风格指南或现有的提示词，为智能体提供背景信息，这在构建新智能体时特别有用。

### 一站式工具管理

我们简化了工具的查看和管理流程。在工作区设置中，你可以集中查看所有已连接的工具、对新工具进行身份验证，并添加远程 MCP 服务器。

如果某个工具需要重新验证，系统会立即提示。连接新工具后，你的所有智能体即可立即使用该工具。为了保证团队的权限治理，只有工作区管理员可以添加新工具，确保工具访问权限始终处于受控状态。

### 立即体验

总结来说，本次更新带来了：可调用所有工具的全局对话、一键将对话转化为智能体的能力、文件上传功能以及更简单的工具管理。这一切都让“从处理问题到让智能体代劳”的过程变得更加轻松。

如果你还没有尝试过 Agent Builder，现在是开始的最佳时机。如果你已经在使用它构建应用，我们非常期待听到你的反馈。
