# 老虎机 和 Vibe Coding 的异同 ​​​

## 文档信息
- 来源：https://baoyu.io/blog/slot-machine-vibe-coding-comparison
- 发布日期：2025-07-08
- 作者：宝玉

## 摘要

**1) 一句话总结**
本文通过类比老虎机和氛围编程（Vibe Coding），揭示了两者在消耗Tokens、产出随机性、用户心理错觉以及最终获利者等方面的惊人相似性。

**2) 关键要点**
*   **消耗机制**：两者均需购买并消耗“Tokens”（赌博代币或AI模型代币）。
*   **随机产出**：老虎机可能中大奖或一无所获；Vibe Coding 输入模糊提示词后，可能生成完美应用，也可能产出无法编译的乱码。
*   **系统反馈**：老虎机通过声光效果刺激玩家；AI系统则通过“绝妙的主意”、“完美解决方案”等热情话语鼓励用户。
*   **掌控错觉**：老虎机玩家自认拥有“赢钱系统”；Vibe Coding 用户则自信地自诩为“提示词工程师”。
*   **赌徒心理**：两者均会让用户陷入“再试一次就能赢回来”或“再试一次提示词就能修复Bug”的自我安慰中。
*   **最终赢家**：老虎机的最终赢家总是庄家；Vibe Coding 的最终赢家总是AI模型厂商。
*   **时间黑洞**：两者都会让人产生轻松达成目标（如中100万大奖或周末建好SaaS）的错觉，但清醒后往往会发现不知不觉中流失了大量时间。

**3) 风险/缺口**
*   **代码质量风险**：Vibe Coding 生成的代码可能充满幻觉且无法编译。
*   **时间成本风险**：可能导致严重的效率低下与时间浪费（例如花费3小时用提示词去生成一个原本只需20分钟就能手写完成的函数）。

## 正文
老虎机：买代币（Tokens），拉动拉杆，可能中大奖，也可能一无所获。灯光闪烁、铃声响起：“大奖！”玩家会自信地说：“我有一套赢钱系统”，然后不断安慰自己：“再玩一次，我一定能赢回来。”但最终，庄家总是赢家。偶尔你会以为赚到了轻松的钱：“我中了100万美元大奖！”可当你清醒过来，不禁困惑：“刚才那4个小时去哪了？”

氛围编程：买 Tokens，写一个模糊的提示词，点击“生成”。你可能得到一个完美无缺的应用程序，也可能是一团充满幻觉、无法编译的混乱代码。系统会热情鼓励你：“绝妙的主意！”、“当然可以！”、“这是为你准备的完美解决方案！”你自信地说：“我是一个提示词工程师”，不断安慰自己：“再试一次提示词，这次肯定能修复这个Bug。”但最终，模型厂商总是赢家。偶尔你感觉自己轻松实现了目标：“我一个周末就搭建了一个完整的SaaS！”但清醒过来却突然困惑：“等等，我竟然花了3小时用提示词生成了一个本来只需20分钟就能写完的函数？”
