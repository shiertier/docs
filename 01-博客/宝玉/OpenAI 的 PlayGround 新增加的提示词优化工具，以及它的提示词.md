## 摘要

OpenAI 在 Playground 中新增了提示词生成与优化功能。本文通过一个“把文本改写成播客脚本”的案例，展示该功能如何把简短需求扩展成包含约束、步骤、输出格式和示例的结构化 Prompt，并据此总结它背后的提示词设计偏好。

可以归纳出的核心规律包括：对简单 Prompt 可以直接重写优化，对复杂 Prompt 尽量保留原结构；强调先推理后结论；要求显式声明输出格式；在需要时补充步骤、示例和注意事项；对结构化任务偏向使用 JSON；知识截止时间为 2023 年 10 月。

## 正文

OpenAI 的 Playground 在 GPT-4o 对话界面里加入了一个帮助用户生成和优化 Prompt 的功能。作者认为这个入口的实际效果不错，尤其适合把原本较短、较散的任务描述整理成模型更容易执行的结构化提示词。

文中给出的测试案例，是把一段文本改写成类似 NotebookLM 风格的播客脚本。作者输入的是一份相对简短的目标描述，重点强调深入探讨、目标听众、结构呈现、主持人与专家的分工、内容中立性，以及要加入更容易记忆的现实例子：

```text
帮助把输入的文本改编成播客脚本，注意突出以下关键点：

- 深入探讨
- 目标听众
- 结构与呈现
- 角色与动态
- 内容规则
- 难忘的例子
```

Playground 生成后的 Prompt 明显更“工程化”：它不仅保留原始意图，还自动补出了更明确的要求说明，并进一步组织出 `# Steps`、`# Output Format` 和 `# Examples` 等段落。例如，它会把“改写成播客脚本”进一步拆成阅读原文、重构内容、安排角色互动、插入案例、控制开头与结尾等可执行步骤。

```text
将输入的文本改编为适合播客脚本的内容，突出以下关键点。

# Steps
1. 阅读并理解输入文本的核心内容和要点。
2. 根据关键点重构文本。
3. 定义并分配主持人和专家的角色。
4. 插入生动的例子和轶事。
5. 保持内容结构清晰，并给出令人难忘的结尾。

# Output Format
- 以段落形式呈现播客脚本。
- 所有对话使用角色名字标示。
```

作者随后根据其披露的内部规则，总结出这个 Prompt 优化器背后有一套相当明确的模板偏好：

- 对简单 Prompt 可以直接增强；对复杂 Prompt 则尽量少改结构，只补齐缺失信息。
- 倾向把“推理过程”放在“结论、分类或结果”之前。
- 会显式要求输出格式，并对结构化任务偏向给出 JSON 一类的机器可解析形式。
- 鼓励使用 Markdown 组织提示词，必要时加入步骤、示例和补充说明。
- 会尽量保留用户已经提供的变量、样例、规则和常量，而不是粗暴重写。
- 文中披露的版本显示，该工具的知识截止时间被设定为 2023 年 10 月。

从这个案例可以看出，Playground 的“提示词优化”并不是简单润色语句，而是在尝试把用户意图改写成更接近提示词工程模板的形式：先明确任务，再补充约束，最后规定输出和示例。这也解释了为什么生成结果通常比用户的原始描述更长、更结构化。
