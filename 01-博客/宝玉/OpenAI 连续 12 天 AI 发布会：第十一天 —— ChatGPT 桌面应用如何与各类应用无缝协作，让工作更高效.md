# OpenAI 连续 12 天 AI 发布会：第十一天 —— ChatGPT 桌面应用如何与各类应用无缝协作，让工作更高效

## 摘要

### 1) 一句话摘要
OpenAI 在其连续12天发布会的第11天推出了 ChatGPT 桌面应用的重大升级，使其能够直接读取并协同 Warp、Xcode、Notion 等第三方应用的内容，大幅提升终端操作、代码编写与文档写作的工作效率。

### 2) 核心要点
*   **上下文感知与无缝协作**：Mac 桌面版 ChatGPT 现可作为独立轻量级窗口运行，通过快捷键（如 Option + Space）快速调用，并能自动获取当前正在使用的第三方应用的上下文。
*   **终端协作（以 Warp 为例）**：支持与 Warp 终端集成，演示中利用 ChatGPT 结合高级数据分析功能，直接读取 Git 提交记录并生成了可视化的柱状图。
*   **IDE 编程支持**：支持 Xcode、VS Code、JetBrains 生态（如 Android Studio、PyCharm）等主流 IDE。演示中调用了 o1 模型在 Xcode 中成功生成并调试了 Swift 辅助功能代码。
*   **文档与写作工具集成**：新增对 Notion、Apple Notes 和 Quip 的支持。演示中 ChatGPT 读取了 Notion 文档的上下文，结合联网搜索（带引用链接）补充了准确的历史事实，并成功模仿了用户的写作风格。
*   **高级语音模式结合**：引入了全新的语音交互方式，用户可以直接通过语音对话与当前打开的文档或代码进行交互（演示中通过“圣诞老人”语音助手修改了 Apple Notes 中的歌单）。
*   **深度内容抓取**：在获得授权的前提下，ChatGPT 不仅能“看到”屏幕上的截图，还能通过 API 抓取应用程序中未显示在屏幕上的完整内容。
*   **发布状态**：Mac 版本已正式上线所有演示功能（用户需更新至最新版），Windows 版本即将推出。
*   **未来产品方向**：OpenAI 计划在 2025 年让 ChatGPT 具备更强的“代理能力（Agentic）”，从单纯的问答转向代表用户执行任务和自动化桌面工作流。

### 3) 风险/不足
*   **代码生成的稳定性**：在 Xcode 的现场演示中，o1 模型首次生成的代码未能按预期运行，需撤销更改并重新生成后才获得成功。
*   **平台可用性限制**：目前所有演示的新功能仅限 Mac 版本可用，Windows 用户暂无法使用（处于“即将推出”状态）。
*   **隐私与授权边界**：ChatGPT 必须在用户明确授权特定应用程序后才能读取其内容，未授权应用的内容无法被访问。
