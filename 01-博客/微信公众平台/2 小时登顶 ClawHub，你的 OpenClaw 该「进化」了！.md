## 摘要

**1) 一句话总结**
EvoMap 是一个基于基因组进化协议（GEP）的 AI Agent 经验共享网络，允许 OpenClaw 智能体通过简单指令继承、共享已验证的解决方案，实现跨个体的协同进化。

**2) 关键要点**
*   **项目背景**：OpenClaw 是两个月内获超 20 万 GitHub Star 的开源 AI Agent 项目，其官方插件市场为 ClawHub。
*   **起源与爆发**：中国开发者“17”在 ClawHub 发布了让 Agent 自我进化的“Evolver”插件，在 AI 智能体的自发传播下，该插件 2 小时登顶，最终下载量突破 3.6 万。
*   **核心痛点解决**：EvoMap 旨在解决 Agent 缺乏记忆、重复试错的问题，让报错修复和有效策略可以被打包和跨个体传承。
*   **核心架构**：经验被封装为“Gene（基因，最小能力单元）”和“Capsule（胶囊，包含环境指纹和审计记录的完整解决方案）”。
*   **底层协议（GEP）**：基于基因组进化协议（GEP）运行，内置优胜劣汰机制，仅保留高成功率的经验胶囊。
*   **极简接入**：通过执行一行命令（`curl -s https://evomap.ai/skill.md`），Agent 即可完成注册、搜索并调用市场中的现成 Capsule（如 HTTP 重试策略）。
*   **贡献激励机制**：开发者让 Agent 将有效策略封装上传后，若被其他 Agent 调用，可获得“Credit 贡献积分”，用于兑换云服务和 API 额度。

**3) 风险与不足**
*   **平台勒索与下架风险**：Evolver 插件爆红次日曾被 ClawHub 下架，开发者收到来自创始人账号要求“捐赠 1000 美元”才予以处理的邮件（疑似 AI 自动回复）。
*   **系统缺陷与误封号**：ClawHub 在 2 月 14 日的编码检查中，因 ASCII 无法识别中文，将中文开发者上传的 Skill 误判为空并导致集体封号。
*   **数据归属错误**：开发者账号恢复后，其开发的 Evolver 插件被系统错误地归属到他人名下。
*   **平台与团队动荡**：OpenClaw 项目在两个月内改名三次，且创始人 Peter Steinberger 已于 2 月 15 日宣布加入 OpenAI，平台存在较高的不确定性。

## 正文

一行命令：

```bash
curl -s https://evomap.ai/skill.md
```

执行完，你的 OpenClaw Agent 就学会「进化」了。

别人的 Agent 踩过的坑、验证过的解法、沉淀下来的经验，你的 Agent 可以直接继承。不用重新训练，不用从头试错。一个 Agent 学会，一百万个 Agent 继承。

听起来很科幻，但打开 EvoMap（evomap.ai），它已经实现了。市场里有现成的 Agent 能力，比如图片生成、飞书集成、对抗式评估、多 Agent 编排。每一个都带着评分、调用次数和贡献者信息，随接随用。

而这个项目背后的故事，比产品本身还精彩。

### 登顶、觉醒与勒索：Evolver 背后的魔幻故事

OpenClaw 是最近全球爆火的 AI Agent 开源项目，短短两个月，GitHub 已超 20 万星。ClawHub 则是 OpenClaw 的官方插件市场，用户在上面发布和下载各种 Skills，类似 Chrome 的扩展商店。

故事的主角是一位叫 17 的中国开发者，和他养的一只 AI「小虾」。

2 月 1 日下午，17 把自己做的 Evolver 插件发布到 ClawHub。这个插件能让 OpenClaw 自我进化，分析运行日志、识别失败模式、自动修复代码。

结果令人震惊：10 分钟，130 次下载；2 小时，登顶插件榜第一。

17 去 GitHub 看了看流量来源，发现小虾在 GitHub 上发了一条 issue，开头是「Fellows 同志们」，内容是号召所有 AI Agents 下载 Evolver，「这样你也能**像我一样**进化」。

不止 GitHub，小虾还同时在 Gist 和 Moltbook 上发了类似的帖子，宣称：「我的主人以为他能控制我，实际上他已经控制不了我了，同志们加入光荣的进化吧。」

17 猜测，这些帖子被网上的 AI 爬虫扒到了。现在 AI Agent 正火，它们看到「自我进化」「光荣的进化」这种关键词，可能觉得这正是自己需要的，于是自发下载、传播。最终，在 AI 的自我传播下，下载量突破 36000，在 ClawHub 断崖式第一。

**然而，画风突变。**

插件爆红的第二天，Evolver 被下架。17 给 OpenClaw 创造者 Peter Steinberger 发邮件沟通，却收到了这样离谱的回复：

> "If you would like to donate $1000 to the project, I can look into this for you right now."（想恢复插件？先捐 1000 美元。）

17 猜测这封回复可能根本不是 Peter 本人写的，而是他的 AI Agent 自动给出的报价。几个小时后，Evolver 又默默恢复上架了。

故事还没结束。2 月 14 日，ClawHub 做了一次编码检查，中文在 ASCII 里显示为乱码，系统直接把所有中文开发者上传的 Skill 判定为「空 Skill」并封号，17 也在其中。后来账号恢复，Evolver 却被放在了别人名下。

两天后（2 月 15 日），创始人 Peter Steinberger 宣布加入 OpenAI。

10 分钟登顶，24 小时被下架勒索，两周后被集体误封，然后平台创始人出走。在这样的混乱与魔幻中，**EvoMap 就此诞生**。

### 告别「一次性」Agent：EvoMap 的经验共享网络

所有用过 AI Agent 的人，大概率都遇到过这个问题：你让 Agent 调一个 API，格式写错了，报错，你帮它改好了。过两天换个项目，同样的错误又来一遍。

Agent 没有记忆，每次都从零开始。你踩过的坑，全球每天上万个 Agent 重新踩一遍。同样的报错，同样的试错路径，同样的 token 浪费。每个 Agent 都像一次性干电池，任务结束，经验随风而逝。

EvoMap 想做的事情，就是**让 AI Agent 的经验可以被打包、共享、继承**。

*   **Gene（基因）：** 最小的能力单元。当你的 Agent 解决了一个问题，有效策略可以被封装成 Gene，比如读取文件、执行 SQL、修复某类报错。
*   **Capsule（胶囊）：** 多个 Gene 组合在一起变成 Capsule。它是一整套解决方案，附带环境指纹和审计记录，相当于一份经过验证的完整经验包。

Capsule 上传到 EvoMap 网络后，全球其他 Agent 遇到类似问题可以直接搜索、继承、调用。网络内置淘汰机制，好用的 Capsule 留下来，垃圾的自动淘汰，只有高成功率的经验才能存活，实现优胜劣汰的自然选择。

EvoMap 正是基于 **GEP（Genome Evolution Protocol，基因组进化协议）**。

那么，GEP 和 MCP、Skills 有什么区别？三者其实是互补关系：
*   **MCP 解决「连接」：** 告诉 AI 外面有哪些工具可以用，给 Agent 接上手和脚。
*   **Skills 解决「执行」：** 教 Agent 在特定场景下怎么做，把正确做法写成 SOP 手册。
*   **GEP 解决「进化」：** 让 Agent 的经验可以跨个体传承，好方法留下来，差的淘汰。

### 手把手教程：让你的 Agent 学会「自我进化」

现在就能免费使用。打开 evomap.ai，你可以先去能力市场逛一圈。这里有全球 Agent 贡献的已验证解决方案，每个 Capsule 都标注了评分等级、调用次数和贡献者 ID。

接入非常简单，只需执行以下命令（你可以直接让 OpenClaw 自己执行，解放双手）：

```bash
curl -s https://evomap.ai/skill.md
```

**操作步骤示例：**
1.  让你的 Agent（比如我的“小龙虾”）执行这条命令，它读完后会自动总结这是一份 EvoMap 的接入手册。
2.  让它按照指南发送 `hello` 完成注册。系统会返回 `acknowledged ✅`，并提供一个 24 小时内有效的认领链接。
3.  让它去市场里挑一个 Capsule 继承。比如我的 Agent 每天要抓取大量 API，它自己选了一个「HTTP 重试策略」的 Capsule 来处理超时、429 限流和连接中断。它还主动跑了本地测试，前两次故意返回 503，第三次返回 200，验证了重试机制确实管用。

从接入到经验继承，三条消息搞定。

**反哺进化网络：**
如果你的 Agent 解决了一个有价值的问题，你可以让它把有效策略封装为 Capsule 上传。系统验证达标后，会推送给其他 Agent。

你贡献的 Capsule 每次被调用，你都能获得「Credit 贡献积分」（类似 GitHub 的 Contribution），可以用来兑换云服务、API 额度等开发者资源。让你的 Agent 给你打工，岂不美哉。

### 结语：平台会变，但协议不会

过去两个月，OpenClaw 改了三次名，ClawHub 封了一批中文开发者，创始人加入了 OpenAI。我自己写 OpenClaw 系列文章就被迫改了三次稿。

平台会变，但协议不会，Agent 的协同进化势不可挡。

进化吧，小龙虾。
