*   **Harness 已开源**：作为指令和工具的集合，Harness 目前已开源并存放在 GitHub 的 `openai/codex` 仓库中。
*   **交互面的作用**：Surfaces 决定了用户与智能体进行交互的具体方式。
*   **原生环境训练**：首次有内部人士证实，Codex 系列模型是直接针对 Codex Harness 环境进行训练的。
*   **深度融合的机制**：工具使用、执行循环、代码压缩与迭代验证等行为是模型学习的内在组成部分，而非后期附加功能。
*   **双向塑造**：Harness 的设计紧密围绕模型如何规划、调用工具及从失败中恢复来展开，两者相辅相成。

## 正文
OpenAI 亚太区开发者体验工程师 Gabriel Chua 近期分享了他对“Codex”这一术语的看法。在 OpenAI 的生态系统中，Codex 这个词往往容易令人混淆，因为它可能指代许多不同的事物。

简而言之，**Codex 是 OpenAI 的软件工程智能体（Agent）**，用户可以通过多种交互界面来使用它。而所谓的“智能体”，本质上就是一个模型加上指令和工具，并封装在一个可以代你执行任务的运行环境中。

### Codex 的三大核心组件

从宏观层面来看，Codex 是由三个协同工作的部分组成的：

*   **Codex = 模型 (Model) + Harness (指令与工具集) + 交互面 (Surfaces)**
*   **模型 + Harness = 智能体 (Agent)**
*   **交互面 = 用户与智能体交互的方式**

Gabriel 将“Harness”定义为“指令和工具的集合”。值得注意的是，这部分内容是开源的，目前存放在 GitHub 的 `openai/codex` 代码仓库中。

### 模型与 Harness 的深度融合

Gabriel 的分享中包含了一个非常关键的信息——这也是首次有 OpenAI 内部人士证实：**Codex 系列模型是直接针对 Codex Harness 进行训练的。**

Codex 模型在训练时就已经置于 Harness 的环境中。这意味着，工具的使用、执行循环、代码压缩（compaction）以及迭代验证，这些都不是后期强加给模型的附加行为，而是模型学习运作方式的内在组成部分。

反过来，Harness 的设计与塑造，也是紧紧围绕着模型如何进行规划、如何调用工具以及如何从失败中恢复来展开的。两者相辅相成，共同构成了强大的软件工程智能体。
