## 原则

文档系统不是一次性搭完就结束。任何一次返工，如果暴露的是结构问题、规则缺口或边界漂移，都应反向沉淀为文档或检查器。

## 最低必跑检查

### 文档系统结构

```bash
python3 scripts/check_docs_system.py
```

### 标题空行

```bash
python3 scripts/normalize_h2_spacing.py --check AGENTS.md skills 00-元语 01-博客 02-资源 03-图书 04-聊天 06-文档系统
```

### 博客发布日期

```bash
python3 scripts/check_recent_blog_publish_dates.py --missing-publish-date 01-博客
python3 scripts/check_recent_blog_publish_dates.py --list-recent 01-博客
python3 scripts/check_recent_blog_publish_dates.py 01-博客 .tmp/publish-date-evidence-final.tsv
```

### 占位词与导航噪音

```bash
rg -n "待补充|TODO 占位|占位文件|## TODO|跳至主要内容|返回主菜单|研究索引|商业应用概览|开发者专区" 01-博客 02-资源 03-图书 04-聊天
```

### 关系图谱有效性

```bash
rg -n "^## 关联主题$|^## 标签$" 00-元语 01-博客 02-资源 03-图书
rg -n "^- #[a-z0-9-]{2,32}$" 00-元语 01-博客 02-资源 03-图书
rg -nP "^- \\[\\[.*\\]\\](?!.*关联理由：)" 00-元语 01-博客 02-资源 03-图书
```

## 园艺触发条件

- 同一类返工在一周内重复两次以上。
- `AGENTS.md` 再次开始承载大量细则，丧失“地图”性质。
- 某个流程必须同时翻多个技能文档才能跑通，说明缺少稳定总览页。
- 首页或站点导航已经无法让新 agent 快速发现规则源头。

## 完成定义

- 规则已写进 `06-文档系统/` 或对应 `skills/*.md`
- 至少新增或更新一条可执行检查
- 检查通过后再发布
