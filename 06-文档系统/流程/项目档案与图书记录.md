## 适用范围

本流程覆盖两类对象：

- `02-资源/` 下的网站 / Git 项目 / 工具 / 平台档案
- `03-图书/` 下的书目记录与读书笔记

对应技能分别是 `skills/site-project-profile.md` 与 `skills/book-record.md`。

## 项目档案

### 固定落盘规则

- 最终文档只落到 `02-资源/{类别}/{中文标题}.md`
- `02-资源/` 一级目录必须是类别，不能是项目名目录
- 标题与文件名必须自解释，推荐结构为 `{对象名}：{对象类型}{核心定位}`

### 写作原则

- 只写可验证的信息，不做脑补，不搬运营销话术。
- 结构优先围绕功能、定位、典型场景、差异点、使用方式概览、限制与注意事项。
- 若 README 不足，可去官方文档、发布说明、包管理器页面或社区讨论补齐，但要区分“官方事实”和“社区经验”。

## 图书记录

### 固定落盘规则

- 最终文档只落到 `03-图书/{作者}/{书名或书名-版本}.md`
- 结构优先写基本信息、内容概览、阅读建议、笔记、链接
- 图书条目的目标是复用与检索，不是复制整本书的正文

### 写作原则

- 只记录确定的作者、版本、译本、出版社、ISBN、阅读建议或自己的笔记。
- 不写 `未识别`、`待补充` 等占位字段。
- 未经允许不搬运大段受版权保护的正文。

## 共同原则

- `title` 和文件名语义必须一致。
- 临时材料和外部抓取结果仍然只放 `.tmp/`。
- 若无法判断对象类型与核心定位，宁可暂缓收录，不写泛化占位标题。
