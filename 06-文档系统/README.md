## 定位

`06-文档系统/` 是 `wsl-docs` 的记录系统入口，用来承载稳定规则、操作流程、维护机制与迁移计划。它的目标不是替代内容库，而是给内容库、技能文档与脚本提供一张可导航的地图。

这次调整参考了 `[[01-博客/OpenAI/Harness Engineering：Agent 优先时代的 Codex 协作|Harness Engineering：Agent 优先时代的 Codex 协作]]` 的核心做法：`AGENTS.md` 只当目录，细节进入结构化文档，规则必须能被机械检查。

## 系统地图

- `AGENTS.md`：最小入口，只保留地图、硬边界与最低自检。
- `06-文档系统/设计/`：稳定原则与结构设计，不放一次性任务说明。
- `06-文档系统/流程/`：按任务类型组织的流程文档。
- `06-文档系统/规则/`：发布口径、输出约束、风险边界。
- `06-文档系统/维护/`：自检、园艺、持续修剪。
- `06-文档系统/计划/`：迁移计划与后续债务。
- `skills/`：战术级技能文档，说明某类任务怎么操作。
- `prompts/`：Gemini 任务提示词模板。
- `scripts/`：可执行脚本与检查器。
- `.tmp/`：临时材料；可以随时清空，不属于最终知识库。

## 渐进式披露路径

1. 从 `AGENTS.md` 进入本目录。
2. 先读一个总览文档，确认当前任务属于哪类流程。
3. 再进入对应 `skills/*.md` 或 `scripts/*`。
4. 只在需要时继续下钻，而不是把全部规则一次性塞进上下文。
