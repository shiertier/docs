## 背景

本轮迁移以 `[[01-博客/OpenAI/Harness Engineering：Agent 优先时代的 Codex 协作|Harness Engineering：Agent 优先时代的 Codex 协作]]` 为参考，目标是把 `wsl-docs` 从“长 `AGENTS.md` + 分散技能文档”调整为“短 `AGENTS.md` + 结构化记录系统 + 机械检查”。

## 本轮已完成

- 新增 `06-文档系统/`，按“设计 / 流程 / 规则 / 维护 / 计划”分层。
- 将 `AGENTS.md` 收缩为入口地图与硬边界。
- 更新 GitBook / Quartz 生成器，把 `06-文档系统/` 纳入站点可见范围。
- 新增 `scripts/check_docs_system.py`，防止结构回退。
- 将 docs-system 检查接入 GitHub Pages 工作流。

## 下一步债务

- 持续把高频但仍然散落在 `skills/*.md` 的规则提炼成更稳定的总览页。
- 评估是否需要增加一页“文档系统质量评分”，跟踪发布日期完整率、关系图谱覆盖率和噪音回归。
- 观察 `06-文档系统/` 的目录粒度是否合适；若某个流程文档再次膨胀，就继续下拆。

## 退出条件

- 新 agent 只读 `AGENTS.md` 和 `06-文档系统/README.md` 就能找到对应流程入口。
- 结构检查可以阻止 `AGENTS.md` 再次长成总手册。
- 首页与站点目录能显式看到“文档系统”入口。
