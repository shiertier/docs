## 定义

dns（Domain Name System）是把域名映射到 IP 地址及其他资源记录的基础互联网命名与解析系统。

## 核心内涵

域名系统（DNS）是互联网的核心基础设施之一，常被比喻为互联网的“电话簿”。它采用分布式、层级化的数据库架构，负责将人类易于记忆的域名（如 `example.com`）转换为机器可使用的资源记录。除了常见的 IP 地址映射，DNS 还承载邮件路由、域名验证、服务发现、流量调度与部分安全控制等关键能力。DNS 的稳定性、缓存策略与授权边界，直接影响网络服务的可达性与变更生效速度。

## 关键机制

- **分层解析链路**：终端通常先向递归解析器发起查询，再由递归解析器沿着根区、顶级域与权威服务器逐级获取最终记录。
- **记录类型体系**：A、AAAA、CNAME、MX、TXT、NS 等记录分别承担地址解析、别名、邮件路由、域名验证与委派等职责。
- **缓存与传播**：DNS 查询结果会被客户端与解析器按 TTL 缓存，因此配置变更往往存在传播窗口，而不是即时全网生效。
- **安全扩展**：DNSSEC 用于校验响应来源与完整性，降低缓存投毒与伪造响应风险。

## 实践要点

- **解析链路理解**：明确递归解析器、根区、顶级域与权威服务器的分工，便于排查解析失败、延迟升高与记录未生效等问题。
- **记录类型管理**：准确配置 A、AAAA、CNAME、MX、TXT 等不同类型的解析记录，以满足网站托管、邮件服务及域名验证等需求。
- **缓存与 TTL 策略**：合理设置生存时间（TTL），在解析生效速度与 DNS 服务器查询负载之间取得平衡。
- **高可用与容灾**：采用多节点、多区域的 DNS 解析服务，防止单点故障导致整个域名的服务中断。
- **安全防护配置**：部署 DNSSEC 以防止 DNS 欺骗与缓存投毒，并利用 DNS 层面进行恶意流量的初步过滤。
