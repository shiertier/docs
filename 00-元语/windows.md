## 定义

Windows 指微软开发的桌面操作系统家族，也是桌面客户端、企业 IT 环境与本地开发工具链的重要宿主平台。在本仓库语境中，它尤其对应“在 Windows 主机上组织本地开发、CLI、WSL 与系统级工具”的一整套工程边界。

## 核心内涵

Windows 不只是一个操作系统名称，它还代表一整套面向桌面计算与企业环境的开发约束，包括 NTFS 文件语义、权限提升与组策略、PowerShell 与 CMD 命令环境、Win32/.NET 等长期演进的系统接口，以及驱动模型、代码签名和系统策略对软件分发与执行的影响。

在本仓库语境中，Windows 的核心价值主要体现在三类场景：一类是桌面客户端、本地自动化与系统级工具；一类是借助 WSL2、Windows Terminal 与容器运行时，把 Linux 工具链引入 Windows 主机，形成混合式开发环境；另一类是需要和企业安全策略、设备权限、文件系统语义直接打交道的本地工程场景。

## 实践要点

- 处理跨平台兼容时，关注路径分隔符、CRLF、大小写敏感差异与编码问题，避免脚本和构建流程只在单一平台可用。
- 区分 PowerShell、CMD 与 WSL 的运行边界，明确命令语法、环境变量、权限模型与文件路径映射是否一致。
- 当项目依赖 Linux 原生命令、容器、文件监听或权限语义时，先确认代码和数据究竟位于 Windows 文件系统还是 WSL 文件系统，避免把跨边界 I/O 与挂载差异误判成应用故障。
- 开发桌面端或系统级工具时，提前评估 UAC、驱动模型、代码签名、企业策略与安装分发机制对功能落地的限制。
- 涉及文件监控、挂载、容器或虚拟化场景时，注意 NTFS 与 WSL 文件系统之间的性能和语义差异。

## 边界与区分

- Windows 与 desktop-client 不同：Windows 是宿主操作系统与权限边界，desktop-client 是运行在其上的应用形态。
- Windows 与 terminal、cli 不同：Windows 可以承载 PowerShell、CMD、WSL 等多种命令环境，但这些交互界面不等于 Windows 本身。
- Windows 与 virtual-file-system、sandbox 不同：前者强调统一文件接口或访问抽象，后者强调执行隔离；Windows 则提供这些机制实际落地时必须面对的底层文件、权限与进程语义。
