## 定义

Durable Objects 是 Cloudflare Workers 提供的有状态执行原语。每个对象都以全局唯一标识对应到单个实例，把请求处理、内存状态与持久化存储绑定在一起，用于在无服务器环境中实现强一致的协调逻辑。

## 核心内涵

- **单实例串行执行**：同一个对象的请求会在单个实例内按顺序处理，适合实现计数器、会话协调、房间状态等需要避免并发竞争的逻辑。
- **计算与存储合一**：对象实例既可保留内存中的热状态，也可把数据写入持久化存储，减少无状态函数与外部数据库之间的往返协调。
- **平台托管路由与生命周期**：平台负责按对象 ID 路由请求、唤醒休眠实例并处理迁移恢复，开发者无需自行实现分布式锁或分片路由。
- **适合实时协作场景**：长连接、房间广播、协作编辑、速率限制和有状态 Agent 运行时，常利用其强一致状态与连接管理能力。

## 实践要点

- **对象边界要收敛**：按用户、房间、租户或会话等天然键拆分对象，避免把全局热点集中到单一对象。
- **把串行语义用在需要协调的地方**：Durable Objects 更适合承担协调层，而不是代替所有业务计算；长时间阻塞会直接拖慢该对象的吞吐。
- **区分热状态与持久状态**：频繁读取的数据可留在内存，但关键状态仍应能从持久化存储重建，避免把实例常驻当成强保证。
- **围绕具体场景落地**：聊天房间、速率限制、任务编排与有状态 Agent 运行时，通常比“把任意服务都塞进对象里”更适合作为切入点。
