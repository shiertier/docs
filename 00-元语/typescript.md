## 定义

TypeScript 是 JavaScript 的语法超集，为 JavaScript 增加可选静态类型、类型推断与更强的语言服务能力。它通常通过 `tsc` 或兼容工具完成类型检查与代码转换，最终仍运行在现有的 JavaScript 运行时中。

## 核心内涵

TypeScript 的核心价值不是发明另一套独立运行时，而是把数据结构、接口边界和约束尽量前移到编辑期与编译期，让错误更早暴露，并提升大型代码库中的可读性、协作效率与重构信心。它通过类型系统、控制流分析、泛型、联合类型和映射类型等能力表达复杂业务模型，同时把 `tsconfig`、语言服务、自动补全、重构与错误提示整合进同一套工程体验中，因此收益通常同时体现在 IDE 反馈和 CI 门禁上。

## 实践要点

- 优先在公共接口、配置对象、数据模型和组件边界定义稳定类型，再逐步收紧内部约束，避免 `any` 蔓延。
- 在 SDK、库和团队协作场景里，应优先保证类型命名清晰、含义稳定，而不是只追求“类型技巧”本身复杂。
- 优先启用 `strict` 等关键编译选项，并把 `tsconfig` 纳入版本控制，避免团队成员各自处在不同的类型约束下。
- 把类型检查同时接入本地 IDE 反馈和 CI 流水线，减少问题只在合并后才暴露。
- 明确区分编译期安全与运行时安全：外部输入、网络响应和用户数据仍需要额外校验，不能把 TypeScript 类型当成运行时验证替代品。

## 边界与区分

- TypeScript 与 JavaScript 不同：TypeScript 扩展的是 JavaScript 的表达与检查能力，不是一个脱离 JavaScript 生态单独运行的新平台。
- TypeScript 与运行时校验不同：类型信息主要服务于编辑期和编译期，编译后不会自动替你验证外部输入是否合法。
- TypeScript 与具体框架不同：React、Node.js、Angular 等都可以把 TypeScript 作为上层开发语言，但它本身不绑定某一个框架或运行时。
