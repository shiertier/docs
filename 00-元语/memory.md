## 定义

memory 指 AI 系统在多轮交互与长期任务中，对历史信息、用户偏好、任务状态与中间结论进行存储、检索、更新与必要遗忘的能力。
memo 在本仓库中视为 memory 的同义表达，不单独设元语词条。

## 核心内涵

在 Agent 与 LLM 应用中，memory 不是单一技术点，而是“状态持续化 + 按需检索 + 写回更新 + 过期淘汰”的组合能力。它决定了系统能否跨会话保持上下文一致性、减少重复输入、支持长期规划与个性化协作。

memory 的关键不只是“存下来”，而是四个问题是否有清晰策略：什么值得记、何时拉回、如何避免噪音、何时删除或遗忘。常见实现会同时混合结构化状态、语义检索片段与任务过程笔记，而不是依赖单一存储形式。

## 与相邻概念的区别

- 与 `context-database` 的区别：memory 更强调系统保留和调用历史信息的能力；context-database 更强调支撑这类能力的存储、检索与生命周期基础设施。
- 与 `RAG` 的区别：RAG 主要描述“先检索、后生成”的调用链路；memory 还覆盖状态写回、长期维护与跨会话延续。
- 与 `knowledge-graph` 的区别：knowledge-graph 是组织长期记忆的一种结构化方式，适合表达实体关系，但不是 memory 的唯一实现形态。

## 实践要点

- 区分短期记忆与长期记忆，避免把全部上下文塞入单轮窗口。
- 结合结构化状态、语义检索与任务笔记，提升召回准确率与可解释性。
- 对记忆写入、合并、淘汰与删除建立明确规则，控制噪音和陈旧信息。
- 只沉淀可复用且与当前目标强相关的信息，避免把临时输出和偶然闲聊全部持久化。
- 在评测中单独验证记忆读写一致性、命中质量与跨任务稳定性。
