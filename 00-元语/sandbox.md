## 定义

sandbox 指把代码、命令或进程放进受控边界内运行的隔离环境，用来限制其对文件系统、网络、系统调用、凭证与宿主资源的访问范围，从而降低误操作、越权与恶意行为的影响。

## 核心内涵

在本仓库语境中，sandbox 通常不是单一技术名词，而是一组执行边界设计的统称。它既可以依托容器、微虚拟机、Wasm、V8 isolate 或操作系统原生机制实现，也包含权限建模、最小权限默认值、失败反馈、审计与提权流程。

对于 Agent 与代码执行场景，sandbox 的关键不只是“隔离起来”，而是让模型在可用性与安全性之间取得平衡：既能完成测试、构建与调试等真实任务，又不能默认拿到超出任务所需的高风险权限。

## 实践要点

- 先定义边界对象：明确被约束的是 shell、代码执行器、浏览器自动化，还是第三方工具调用，避免把局部隔离误当成全局安全。
- 把文件系统、网络、系统调用与凭证访问拆开建模，分别决定默认拒绝、显式放行和提权条件。
- 当沙盒拦截正常任务时，要同时给出清晰的失败原因与提权路径，否则 Agent 容易盲目重试或诱导用户过度放权。
- 选择实现方案时，不只看隔离强度，还要一起评估冷启动、兼容性、可观测性与调试成本。

## 边界与区分

- sandbox 与 security 不同：sandbox 是执行边界与隔离机制，security 是更广泛的整体安全能力。
- sandbox 与 risk 不同：risk 负责识别和分级不确定性，sandbox 负责把其中一部分高风险操作收敛到可控边界内。
- sandbox 与 terminal、cli 不同：terminal 和 cli 是交互载体或接口，sandbox 决定其中命令与工具能拿到哪些权限。
- sandbox 与 virtual-file-system 不同：virtual-file-system 统一文件语义，sandbox 关注是否允许访问以及访问时的隔离与约束。
