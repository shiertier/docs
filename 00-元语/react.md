## 定义

React 是用于构建 Web 与原生用户界面的 JavaScript UI 库，核心方式是用组件组合界面，并通过 props、state、context 与 Hooks 组织数据和交互逻辑。它本身不是完整前端技术栈，但常作为 Next.js、React Native 等上层框架或渲染方案的基础。

## 核心内涵

React 的重点不是直接命令式操作 DOM，而是先把界面拆成组件，再用最小必要的 state 描述会变化的数据，让 UI 随状态变化而重新计算。它强调单向数据流、组件组合和可复用逻辑边界；Hooks 则让状态、副作用与上下文能力能够按组件结构被组织和复用。对 React 的理解若只停留在“虚拟 DOM 优化性能”，往往会忽略它更核心的状态建模与组件化抽象。

## 实践要点

- 先按数据结构和交互边界拆组件，再决定组件层级与职责，不要先堆页面再被动抽象。
- 只保存最小必要的 state；能从现有 props 或 state 推导出的数据，不要重复存储。
- 优先保持单向数据流；多个组件共享状态时，先考虑上移 state，再考虑 context 或专门状态方案。
- 把副作用、异步请求和 DOM 访问限制在明确边界内，避免在渲染阶段写命令式逻辑。
- 把 React 视为 UI 基础层；路由、SSR、构建、样式和原生端能力通常依赖 Next.js、Vite、React Native 等周边工具补齐。
