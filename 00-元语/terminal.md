## 定义

terminal 指承载字符式输入输出的交互环境；在现代开发语境中，通常表现为终端模拟器、远程伪终端或 Web 终端中的会话窗口，是 CLI、Shell、REPL 与各类命令行工具运行的主要载体。

## 核心内涵

terminal 的核心不是“命令本身”，而是提供一段可持续的文本交互会话：它承载标准输入输出、当前目录、环境变量、会话状态与快捷键操作，并把用户输入交给 Shell 或具体程序处理。同一个 terminal 会话既可以运行传统 Shell 命令，也可以承载数据库客户端、语言 REPL、全屏 TUI 程序或 terminal-first Agent。在本仓库语境下，terminal 是开发、调试、部署、远程连接与自动化执行的基础工作面。

## 实践要点

- 执行命令前先确认当前目录、环境变量与目标环境，避免把本地、远程、测试、生产会话混用。
- 对高频操作可结合别名、历史记录与补全机制提效，但不要让隐式状态掩盖真实执行上下文。
- 长任务、远程会话或多项目并行时，优先使用多路复用工具维持会话连续性与日志可追溯性。
- 涉及删除、覆盖、提权或批量修改时，应显式做二次确认，并遵守最小权限原则。

## 边界与区分

- terminal 与 cli 不同：terminal 是交互环境与会话载体，cli 是程序暴露出来的命令式接口。
- terminal 与 tool 不同：terminal 是通用运行平面，tool 是在其中被调用的具体能力单元。
- terminal 与 workflow 不同：terminal 可承载多个步骤，但 workflow 关注的是步骤编排、依赖关系与执行治理。
- terminal 与 desktop-client 不同：很多终端模拟器以桌面客户端形式出现，但 terminal 这个概念强调的是会话与 I/O 承载能力，而不限于某个本地 GUI 应用。
