## 定义

Codex 是 OpenAI 当前面向编码与任务执行场景的智能体产品族与运行框架统称，强调工具调用、代码执行与工程闭环。在本仓库语境中，它通常不单指某一代代码模型，而是指围绕 CLI、IDE、Web 等入口展开的 coding agent 体系。

## 核心内涵

Codex 的核心不只是“生成代码”，而是把自然语言目标转成可执行的工程流程：理解任务、规划步骤、调用工具、运行命令、读取结果、继续修正，直到产出可交付的软件变更。
在工程实现层面，Codex 以智能体循环和上下文管理为基础，通过 Responses API 驱动推理与工具调用，并常通过 App Server（JSON-RPC over stdio）向 CLI、IDE、Web 与 TUI 暴露能力。MCP 可作为接入方式之一，但通常只覆盖通用能力子集，难以完整表达 Codex 特有的会话语义与执行状态。

## 实践要点

- 关注点不仅是“写代码”，还包括规划、执行、验证、修复与交付。
- 在多 agent 场景中，Codex 常作为 terminal-first 的工程执行体。
- 使用时需提供受控的工具权限、沙箱隔离与审查流程，并结合 CI 降低执行风险。
