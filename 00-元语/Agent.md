## 定义

Agent 指能够围绕目标持续执行“感知环境、做出决策、调用工具、依据反馈继续行动”的 AI 系统。它通常基于 LLM，但不等同于单轮问答模型，也不等同于把固定步骤串起来的 workflow。

命名约定：元语统一使用单数 `Agent`；英文复数 `agents` 仅作为正文自然语言，不作为独立元语词条。

## 核心内涵

Agent 的核心不在“会聊天”，而在“能闭环执行”：

- 围绕目标持续推进，而不是只生成一次性回答。
- 能调用外部工具或系统，对环境产生可验证的影响。
- 能维护状态，在多轮执行中保留上下文、阶段结果与约束。
- 能根据执行反馈调整路径，而不是机械重复预设步骤。

与 workflow 的关键差异在于：workflow 更适合步骤稳定、分支较少的任务；Agent 更适合需要根据外部反馈动态决定下一步的任务。

## 实践要点

- Agent 的关键在于“循环执行 + 工具协作 + 约束控制”。
- 不是所有任务都值得 Agent 化；若流程固定且结果易校验，优先用 workflow 通常更稳。
- 评估 Agent 需要同时看结果质量、过程合规、自主性边界与稳定性。
- 在工程实践中，清晰的提示约束、可靠的工具接口、可观测反馈与必要的人类兜底，决定了 Agent 是否可控。
