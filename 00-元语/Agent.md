# Agent

## 文档信息

- 类型：元语词条
- 更新日期：2026-02-22

## 定义

Agent 指具备目标驱动、工具调用、状态维护与多步执行能力的 AI 系统，不等同于单轮问答模型。

命名约定：元语统一使用单数 `Agent`；英文复数 `agents` 仅作为正文自然语言，不作为独立元语词条。

## 核心内涵

智能体的核心在于其自主性与环境交互能力。它能够理解复杂指令，将其拆解为可执行的子任务，并通过调用外部工具（如 API、终端、数据库）来获取信息或改变环境状态。同时，Agent 具备记忆机制，能够在多轮交互中维持上下文连贯性，并根据执行反馈进行自我纠错与路径调整。

## 实践要点

- Agent 的关键在于“循环执行 + 工具协作 + 约束控制”。
- 评估 Agent 需要同时看结果质量、过程合规与稳定性。
- 在工程实践中，设计清晰的系统提示词、提供可靠的工具接口以及建立完善的异常处理机制是保障 Agent 稳定运行的基础。
