## 定义

Markdown 是一种面向可读性与结构表达的轻量级标记语言，用纯文本描述标题、列表、链接、引用和代码块等常见文档结构，常作为 README、知识库、博客与文档即代码工作流的基础格式。

## 核心内涵

Markdown 的价值不只在于“写起来简单”，更在于它把内容结构、协作审阅与多端发布放在同一个纯文本载体里。人可以直接阅读源文件，Git 可以追踪差异，渲染器又能把同一份内容输出为网页、文档站、笔记或发布页面。因此，Markdown 经常充当知识库、GitHub 协作、静态站点和 AI 工作流中的中间表示层，而不是单一编辑器专属格式。

## 实践要点

- 语义优先：使用标题、列表、引用、表格和代码块表达结构，不用视觉缩进替代语义。
- 兼容性边界：优先采用通用 Markdown 语法；使用 frontmatter、Mermaid、任务列表、脚注等扩展前，先确认目标渲染环境是否支持。
- 文档即代码：把 Markdown 文件纳入 Git 审查、差异对比和自动检查流程，让文档演进与代码变更同步治理。
- 元信息分层：把发布日期、作者、来源等元数据放入 frontmatter，把正文内容放入正文结构，避免把文档说明、导航噪音与内容本体混写。
- 发布一致性：同一份 Markdown 在 GitHub、wiki、静态站点和本地笔记工具中的渲染差异需要提前验证，尤其是链接、代码块、表格与嵌入语法。
