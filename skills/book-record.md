# Book Record

## 目标

把“一本书”整理成一条可复用的图书条目，便于后续检索、选读、复盘与积累笔记：

- 这本书是谁写的/谁翻译的？你记录的是哪个版本/译本？
- 这本书大致讲什么（只在你有可靠来源/读书笔记时写）？
- 阅读门槛与建议路线是什么？
- 你的笔记与摘录是什么？

强调：这里的产出不是搬运图书正文，更不是生成虚构摘要；只写“确定信息 + 你的笔记”。

## 输出位置与命名（强制）

- 输出目录：`03-图书/{作者}/`
- 单文件交付：`{书名}-{版本或译本}.md`（尽量短；无版本可只用书名）
- 临时文件：统一写到当前目录 `.tmp/`（可随时清空；不进入最终结构）

## 固定结构（推荐）

按需增删小节；没有内容就不要写该小节，禁止留空占位。

- `# 书名：版本/译本`
- `## 文档信息`
  - 对象：图书
  - 来源：用户提供 / 书封 / 目录页 / 出版社页 / ISBN 页面等（只写你确实用到的来源）
  - 记录日期：YYYY-MM-DD
- `## 基本信息`（只写确定值）
  - 作者
  - 译者/编者（如适用）
  - 版本/译本/卷册（如适用）
  - 出版社/出版年/ISBN（可选；必须可验证才写）
- `## 内容概览`（可选）
- `## 阅读建议`（可选）
- `## 笔记`（可选：只写你自己的笔记）
- `## 链接`（可选：必须来源可验证）

## 信息来源与边界（硬性）

- 不写 `未识别`、`如识别到` 等占位语；不确定就删字段。
- 禁止在标题/字段名里加括号式说明（例如“作者（如识别到）”）。
- 未经允许不要搬运受版权保护的大段正文：
  - 可以写你自己的总结、评价、阅读笔记。
  - 摘录必须少量、必要，且尽量注明来源（页码/章节）——如果你掌握该信息。
- 若只有书名与作者等书目信息：条目只做书目卡片，不要硬写内容摘要。

## SOP（建议流程）

1) 收集最少但可靠的来源
   - 用户提供信息
   - 书封/版权页（出版信息）
   - 出版社/图书馆/ISBN 页面（用于核对版本）
   - 你的读书笔记（用于内容概览与摘要）

2) 写条目（先写确定信息）
   - 先完成 `## 文档信息`、`## 基本信息`
   - 有读书笔记再写 `## 内容概览`、`## 笔记`、`## 阅读建议`

3) 如存在足够笔记，再生成 `## 摘要`
   - 对你写好的条目正文运行：`scripts/gemini_task.py summarize`
   - 注意：Gemini 只能基于你提供的文本生成摘要，不能补充事实

## 发布前自检

```bash
rg -n "正文（|站点中文链接（|（中文翻译）|（如识别到）|未识别|（提取）" 03-图书
```
